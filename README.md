# 学习理解的步骤

**eg: 设备树**

##   抓住本质

核心问题：“它到底是什么？解决了什么根本问题？”

##   理解演进逻辑

核心问题：“设备树出现过去是怎么做的？为什么不行？它如何改进？”

为什么变”比“变成什么样”更重要。

##  建立评估框架

核心问题：“它有什么好处？适用边界在哪？有没有替代方案？”

未来能评估“是否该用（什么时候用）设备树”或“是否有更好替代”。

比如：设备树不适合动态热插拔设备（所以 USB/PCI 仍靠枚举），这就是边界。
##   形成可迁移的认知模型

核心问题：“这个模式能否抽象成通用原则？用在其他领域？”

配置文件 vs 硬编码

插件系统 vs 内置功能

声明式 API vs 命令式调用
