#   Drivers

先介绍下 **内核模块** 。

Linux 采用的是宏内核架构。宏内核的特点是将操作系统的大部分核心功能（如进程管理、内存管理、文件系统、网络协议栈以及设备驱动等）都放在内核空间中运行，形成一个整体。其优点是执行效率高（系统调用和子系统之间的通信开销较小），但也有一些缺点：
* (1)大量功能静态集成在内核内部，导致：
    * 内核映像臃肿：大量驱动、文件系统和协议都静态编入内核，会使内核变得庞大臃肿。
    * 灵活性，扩展性较差：某些功能或驱动不用时，内核功能难以按需裁剪；新硬件或新功能需求时需要重编整个内核，扩展成本较高。
* (2)大量功能都运行在内核态，导致：
    * 故障影响范围大
    * 安全风险集中
    * 隔离性差

为了缓解其中 **(1)** 类问题，把某个功能单独写成一段代码，在内核运行过程中动态加载到内核中，需要时加载，不需要时卸载.相当于给宏内核增加了“可插拔能力”。这，就是Linux中的 **内核模块（Loadable Kernel Modules, LKM）** 机制。


**Tips:**

在 Linux 里，设备驱动通常不是直接写进内核本体，而是以 内核模块（Kernel Module）的形式存在。

内核裁剪：根据目标硬件和使用场景，对内核功能进行取舍，决定哪些功能直接编进内核、哪些功能编成模块、哪些功能完全不编译，从而让内核更小、更合适、更高效。

注意区分内核模块动态加载不是用户程序的动态链接，动态链接仅限于用户态程序。
